# 安全政策

瞎芝麻重视负责任的安全研究。项目包含管理员鉴权、服务端接口、第三方行情访问、缓存和预计算流程，请勿在公开 Issue 中披露尚未修复的漏洞或可利用细节。

## 支持范围

安全修复优先应用于默认分支 `main` 和最新公开版本。较早的提交、个人 Fork、非官方部署及未经维护者确认的修改版本不保证获得安全更新。

## 私密报告漏洞

请通过 GitHub 仓库的 **Security → Report a vulnerability** 提交私密报告：

<https://github.com/Iman-GGG/xiazhima/security/advisories/new>

如果该入口暂时不可用，请只创建不含敏感细节的普通 Issue，请求维护者提供私密联系方式；不要附加利用代码、真实令牌或生产环境数据。

## 报告内容

高质量报告应尽量包含：

- 漏洞类型和受影响组件；
- 受影响的提交、版本、路由或配置；
- 最小化、无破坏性的复现步骤；
- 预期行为与实际行为；
- 潜在影响和攻击前提；
- 建议修复方案（如有）；
- 是否已在其他地方公开。

请使用测试数据并移除密码、Cookie、会话令牌、环境变量值、个人信息和真实交易数据。

## 我们的处理方式

维护者将尽力：

1. 确认收到报告并进行初步分类；
2. 在可复现后评估影响和修复优先级；
3. 通过私密渠道协调修复与披露时间；
4. 在修复可用后发布说明，并在合适时致谢报告者。

本项目由个人维护，无法承诺固定响应时限，但会优先处理可能导致未授权管理操作、凭据泄露、远程代码执行、敏感数据暴露或供应链风险的问题。

## 安全基线

- 管理员功能要求显式配置 `ADMIN_PASSWORD` 和 `ADMIN_SESSION_SECRET`；缺失时登录保持关闭。
- 生产环境管理员会话 Cookie 默认使用 `HttpOnly`、`SameSite=Lax` 和 `Secure`。
- `.env`、密码、令牌和部署凭据不得提交到仓库。
- 外部输入、股票代码和管理操作应经过校验与鉴权。
- 第三方行情接口响应应视为不可信外部数据，并采用超时、错误处理和缓存策略。

## 不属于安全漏洞的情况

以下内容通常应作为普通 Bug 或数据问题报告：

- 第三方行情延迟、缺失或与其他平台口径不同；
- 固定规则产生的信号与个人判断不一致；
- 在线演示偶发不可用或触发第三方接口限流；
- 不涉及权限绕过或数据暴露的界面与文档问题。

## 安全测试边界

未经明确授权，请勿对公开演示执行拒绝服务、批量扫描、社会工程、凭据尝试、数据破坏或可能影响第三方行情服务的高频请求。请优先在自己的本地环境或 Fork 中进行无破坏性验证。
