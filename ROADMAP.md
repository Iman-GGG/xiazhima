# 项目路线图

本路线图用于公开瞎芝麻的维护方向，不代表交付承诺。优先级会根据安全、数据源变化、可复现问题和维护能力调整。

## 已具备

- [x] Next.js 全栈裁断台与公开在线演示；
- [x] B1、B2、单针下三十、S1 与大势规则实现；
- [x] 蓝筹、中盘和全 A 等筛选范围；
- [x] 个股搜索、指标拆解与 K 线展示；
- [x] 收盘后预计算与内存、文件、KV 缓存；
- [x] 管理员 OAMV 录入和手动预计算；
- [x] 核心指标与规则单元测试；
- [x] 公开许可证、项目说明、贡献指南和安全政策。

## 近期：开源维护基础

- [ ] 增加 GitHub Actions，自动运行安装、测试、类型检查、Lint 和构建；
- [ ] 提供 Bug、功能建议、安全相关提示与 Pull Request 模板；
- [ ] 增加环境变量示例，确保示例不含任何真实凭据；
- [ ] 补充部署、缓存和预计算故障排查文档；
- [ ] 扩展管理员鉴权、输入校验和 API 错误路径测试；
- [ ] 建立可追溯的版本发布说明。

## 中期：可靠性与可验证性

- [ ] 为 B1、B2、单针下三十和 S1 增加更多边界与回归样本；
- [ ] 为规则版本、阈值和数据日期提供更清晰的页面标识；
- [ ] 改进第三方行情超时、限流、降级和缓存可观测性；
- [ ] 为预计算结果增加一致性检查和可复现诊断信息；
- [ ] 改善移动端、键盘操作、对比度和屏幕阅读器体验；
- [ ] 完善第三方数据来源、适用条款和失败边界说明。

## 长期探索

- [ ] 设计可替换的数据源适配层，降低单一第三方接口依赖；
- [ ] 支持规则版本化和历史样本离线回放，便于验证实现变更；
- [ ] 在不上传用户交易数据的前提下探索本地化研究工具；
- [ ] 建立面向规则、数据和发布产物的供应链完整性检查。

长期探索项目只有在具备明确需求、合规边界、维护资源和测试方案后才会进入开发。

## 明确不做

- 不连接券商账户，不代替用户下单；
- 不提供自动交易、跟单或收益承诺；
- 不将规则信号包装为确定性价格预测；
- 不收集或公开用户的真实持仓与交易记录；
- 不用未经审核的生成式内容直接替代风险提示或投资判断。
